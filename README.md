# Приклад запиту
# URL: POST http://localhost:5000/search
# Content-Type: application/json
# Тіло запиту (JSON)
```json
{
  "items": [
    { "subreddit": "r/nature", "keywords": ["forest", "river"] },
    { "subreddit": "r/aww", "keywords": ["cat","dog"] }
  ],
  "limit": 2,
  "file":false,
  "filterBy":"title",
}
```
# Опис використання
Для кожного переданого сабреддіту сервіс виконує запит до Reddit, отримує список постів та аналізує їхній вміст.

Пости фільтруються за ключовими словами, вказаними у полі keywords.
Поле filterBy визначає, де саме виконувати пошук ключових слів:
title — пошук у заголовку поста
text — пошук у тексті поста
Поле limit обмежує кількість постів, які завантажуються з Reddit для кожного сабреддіту.
Якщо limit не передано або він дорівнює 0, використовується значення за замовчуванням.
Запит і відповідь передаються у форматі JSON або файлом JSON.

# Відповідь на теоретичне питання
Основна проблема такого підходу в тому, що HTML-структура сайту може змінитися, і парсер перестане працювати, але є ще проблеми з policy сайту (можливе блокування запитів). 
Тому, якщо є можливість, краще отримувати дані через API, бо це стабільніше та надійніше.
