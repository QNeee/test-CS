*** Інструкція по запуску ***
BASH:
====================
1. git clone <url_репозиторію>
2. cd test-CS/Test/Test-cs
3. docker build -t test_cs_app .
4. docker run --name test_cs_container -p 5000:5000 -e SERVER_URL=http://*:5000/ test_cs_app
====================

*** Приклад запиту ***
URL: POST http://localhost:5000/search
Заголовки:
====================
Content-Type: application/json
====================

*** Тіло запиту (JSON) ***
====================
{
  "items": [
    { "subreddit": "r/nature", "keywords": ["forest", "river"] },
    { "subreddit": "r/aww", "keywords": ["cat","dog"] }
  ],
  "limit": 2
}
====================

*** Опис використання ***
====================
API імітує роботу Reddit, використовуючи локальні дані (data.json) для тестування.
Пости фільтруються за ключовими словами, зазначеними у keywords.
Поле limit обмежує кількість постів, які повертаються для кожного сабреддіту.
Запит і відповідь формуються у форматі JSON.
====================

*** Відповідь на теоретичне питання ***
====================
Основна проблема такого підходу в тому, що HTML-структура сайту може змінитися, і парсер перестане працювати, але є ще проблеми з policy сайту (можливе блокування запитів). 
Тому, якщо є можливість, краще отримувати дані через API, бо це стабільніше та надійніше.
====================
